# Git Guidelines — postfix-admin-rs

## Branch Model

### Main Branches

| Branch | Role | Protection |
|--------|------|-----------|
| `main` | Stable production | Protected, merge via PR only |
| `develop` | Feature integration | Protected, merge via PR |

### Working Branches

| Prefix | Usage | Example |
|--------|-------|---------|
| `feature/` | New feature | `feature/domain-crud` |
| `fix/` | Bug fix | `fix/login-session-timeout` |
| `refactor/` | Refactoring without functional change | `refactor/repository-traits` |
| `docs/` | Documentation only | `docs/api-endpoints` |
| `chore/` | Maintenance, CI, dependencies | `chore/update-sqlx-0.8` |
| `release/` | Release preparation | `release/v1.0.0` |
| `hotfix/` | Urgent fix on main | `hotfix/sql-injection-fix` |

### Workflow

```
feature/domain-crud ──PR──▶ develop ──PR──▶ main
                                              │
hotfix/critical-fix ──────────────────PR──▶ main
                                              │
                                     tag: v1.0.1
```

## Commits

### Message Format

```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

### Types

| Type | Description |
|------|-------------|
| `feat` | New feature |
| `fix` | Bug fix |
| `docs` | Documentation |
| `style` | Formatting (no code change) |
| `refactor` | Refactoring (no functional change) |
| `perf` | Performance improvement |
| `test` | Test addition or modification |
| `chore` | Maintenance (CI, dependencies, build) |
| `security` | Security fix |

### Scopes

| Scope | Crate/Module |
|-------|-------------|
| `core` | postfix-admin-core |
| `db` | postfix-admin-db |
| `auth` | postfix-admin-auth |
| `api` | postfix-admin-api |
| `web` | postfix-admin-web |
| `cli` | postfix-admin-cli |
| `server` | postfix-admin-server |
| `ci` | CI/CD |
| `deps` | Dependencies |

### Examples

```
feat(db): add PostgreSQL repository for domains

Implements PgDomainRepository with CRUD operations.
Queries use sqlx::query_as! for compile-time verification.

Closes #42
```

```
fix(auth): fix transparent bcrypt password rehashing

The $2y$ prefix wasn't detected correctly, causing login failures
for users migrated from PostfixAdmin PHP.
```

```
chore(deps): update axum 0.7 → 0.8

BREAKING CHANGE: State extractors must use the new API.
```

### Rules

- Subject is max 72 characters
- Subject uses imperative present ("add", not "added" or "adding")
- No period at the end of subject
- Body explains **why**, not **what** (the diff shows what)
- Reference issues with `Closes #N` or `Refs #N`
- One commit = one logical change (no squash commits)

## Pull Requests

### PR Checklist

```markdown
## Description
<!-- What does this PR do and why? -->

## Type of Change
- [ ] New feature (feature)
- [ ] Bug fix (fix)
- [ ] Refactoring
- [ ] Documentation
- [ ] Maintenance / CI

## Checklist
- [ ] Code compiles without warnings (`cargo build`)
- [ ] Tests pass (`cargo test`)
- [ ] Clippy passes (`cargo clippy -- -D warnings`)
- [ ] Formatting is correct (`cargo fmt --check`)
- [ ] Documentation is updated if necessary
- [ ] Migrations are reversible
- [ ] No secrets or credentials in code

## Tests
<!-- How to test this PR? -->
```

### PR Rules

- One PR per feature or fix
- Title follows same format as commits
- At least one reviewer required
- All CI checks must pass
- Squash merge to `develop` (clean history)
- Merge commit to `main` (preserve history)
- Branches are deleted after merge

## Tags and Releases

### Semantic Versioning (SemVer)

```
MAJOR.MINOR.PATCH

1.0.0  → First stable release
1.1.0  → New feature (backward compatible)
1.1.1  → Bug fix
2.0.0  → Breaking change
```

### Release Process

```bash
# 1. Create the release branch
git checkout -b release/v1.2.0 develop

# 2. Update versions in Cargo.toml
# 3. Update CHANGELOG
# 4. Test

# 5. Merge to main
# 6. Tag
git tag -a v1.2.0 -m "Release v1.2.0"

# 7. Merge to develop (to sync)
# 8. Push tags
git push origin v1.2.0
```

## Git Hooks (pre-commit)

```bash
#!/bin/sh
# .githooks/pre-commit

set -e

echo "=== Checking formatting ==="
cargo fmt --check

echo "=== Clippy ==="
cargo clippy -- -D warnings

echo "=== Quick tests ==="
cargo test --lib

echo "All checks passed."
```

Installation: `git config core.hooksPath .githooks`
